Indexed-Dependent Refinements

We illustrate how to use abstract refinements to talk about indexed-dependent invariants.

We use a Vector of as implemented as a function from Int to as

16: data Vec a = V (Int -> a)

We abstract over the dom that describes the domain and rng that describes the value with respect to its index.

23: {-@
24: data Vec a <dom :: Int -> Prop, rng :: Int -> a -> Prop>
25:      = V {a :: i:Int<dom> -> a <rng i>}
26:   @-}

With this we can describe interesting vectors:

A vector of Int defined on values less than 100 containing values equal to their index:

34: {-@ type IdVec = 
35:      Vec <{\v -> (v < 100)}, {\j v -> (v = j)}> Int
36:   @-}

A vector defined on the range [0..n) with its last element equal to 0:

41: {-@ type ZeroTerm N = 
42:      Vec <{\v -> (0 <= v && v < N)}, {\j v -> (j = N - 1 => v = 0)}> Int
43:   @-}

Or a vector defined on integers whose value at index i is either 0 or the ith fibonacci:

49: {-@ measure fib :: Int -> Int @-}
50: {-@ type FibV = 
51:      Vec <{\v -> 0=0}, {\j v -> ((v != 0) => (v = fib(j)))}> Int @-}
